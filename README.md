# X-ponential

## Por que existimos

Hay equipos que nacen alrededor de una tarea. Otros nacen alrededor de una conviccion: que la combinacion correcta de personas, disciplina y confianza puede producir una capacidad que ninguno tendria por separado.

X-ponential parte de esa segunda posibilidad.

Este repositorio existe para darle forma a esa conviccion. Aqui vamos a pensar con rigor, registrar lo importante, contrastar ideas, tomar decisiones y convertir oportunidades en trabajo concreto.

No estamos creando documentos por orden. Estamos creando una memoria compartida para que el avance del equipo no dependa de impulsos, conversaciones dispersas o entusiasmo momentaneo.

## Lo que buscamos

Queremos desarrollar una capacidad colectiva: encontrar oportunidades, evaluarlas con honestidad, construir soluciones utiles y sostenerlas hasta que produzcan valor real.

El resultado podra tomar distintas formas: productos, servicios, experimentos, alianzas, investigacion, empresas o aprendizajes descartados a tiempo. Algunas iniciativas buscaran flujo de caja temprano; otras buscaran construir activos de largo plazo; otras podran responder a una vocacion de impacto aunque no moneticen de inmediato.

## Principios iniciales

- Primero el porque: ninguna iniciativa deberia avanzar solo porque es interesante; debe tener una razon suficiente.
- Valor antes que captura de valor: el retorno economico debe ser consecuencia de resolver algo real.
- Claridad compartida: lo que no se entiende en equipo dificilmente se ejecuta bien en equipo.
- Criterio antes que consenso facil: preferimos conversaciones honestas a acuerdos fragiles.
- Servicio como forma de liderazgo: liderar es hacer que otros puedan ver, decidir y avanzar mejor.
- Evidencia progresiva: una idea gana derecho a mas inversion cuando muestra aprendizaje, traccion o valor.
- Ejecucion sobria: ciclos pequenos, responsables visibles y compromisos verificables.

## Flujo sugerido

1. Registrar una conversacion o necesidad en `docs/reuniones`.
2. Convertir ideas relevantes en una propuesta dentro de `docs/propuestas`.
3. Discutir la propuesta en equipo usando issues, comentarios o pull requests.
4. Registrar decisiones relevantes en `docs/decisiones`.
5. Ejecutar en ciclos cortos con responsables y fechas visibles.

## Participacion inicial

El kickoff no busca cerrar la identidad del equipo. Busca abrir una conversacion ordenada para que cada integrante pueda revisar, cuestionar, completar y apropiarse de lo que este repositorio propone.

La primera invitacion para cada integrante es:

- Revisar el [por que](docs/PORQUE.md).
- Comentar los [principios no negociables](docs/PRINCIPIOS.md).
- Completar o ajustar su perfil en [equipo](docs/EQUIPO.md).
- Proponer una primera oportunidad, tarea o pregunta usando issues.

## Estado actual

Este repositorio esta en fase fundacional. Su primera funcion no es verse terminado; es ayudarnos a formular mejores preguntas, ordenar conversaciones y producir los primeros acuerdos serios.
