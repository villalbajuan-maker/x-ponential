# 0001 - Usar GitHub como memoria operativa

## Estado

Propuesta inicial

## Fecha

2026-06-21

## Contexto

El equipo esta en una etapa temprana: todavia no necesita una organizacion pesada, pero si necesita una forma comun de preservar contexto, formular propuestas y evitar que las decisiones queden dispersas.

GitHub ofrece un entorno conocido para el equipo, permite versionar documentos, discutir cambios y mantener trazabilidad sin introducir herramientas adicionales.

## Decision

Usaremos este repositorio como memoria operativa durante la etapa fundacional.

Aqui registraremos:

- Fundamentos y principios del equipo.
- Acuerdos operativos.
- Propuestas.
- Decisiones relevantes.
- Minutas de reuniones.
- Preguntas abiertas y aprendizajes.

## Consecuencias

- La informacion importante deberia quedar disponible para todos.
- Las propuestas tendran una estructura minima antes de consumir tiempo del equipo.
- Las decisiones podran revisarse con contexto, no solo por memoria.
- El repositorio debe mantenerse ligero; si se vuelve burocratico, habra que ajustar el proceso.

## Pendiente

- Definir nombre definitivo del equipo o iniciativa.
- Completar integrantes, fortalezas y disponibilidad.
- Definir ritmo de reuniones.
- Acordar como se usaran issues, branches y pull requests.
