# Acuerdos de colaboracion

Este documento recoge acuerdos iniciales para trabajar juntos. Debe cambiar cuando la experiencia nos ensene una mejor forma.

## Fuente de verdad

El repositorio sera la fuente principal para:

- Propuestas.
- Decisiones relevantes.
- Minutas de reuniones.
- Criterios de priorizacion.
- Aprendizajes del equipo.

Los chats pueden servir para conversacion rapida, pero lo importante debe aterrizar aqui.

## Forma de proponer

Una propuesta debe explicar:

- Que problema u oportunidad aborda.
- Por que importa ahora.
- Que alternativa concreta plantea.
- Que recursos requiere.
- Que riesgos o incertidumbres existen.
- Cual seria el primer paso ejecutable.

La plantilla base esta en [docs/propuestas/TEMPLATE.md](propuestas/TEMPLATE.md).

## Forma de decidir

No toda decision necesita documento formal. Si una decision cambia direccion, compromete tiempo relevante, define inversion, crea una regla de trabajo o afecta a todo el equipo, debe quedar registrada en `docs/decisiones`.

Para decidir, procuramos responder:

- Que estamos decidiendo.
- Que opciones consideramos.
- Que criterio usamos.
- Quien queda responsable de ejecutar o dar seguimiento.
- Cuando revisaremos si funciono.

## Reuniones

Cada reunion importante debe dejar una minuta breve en `docs/reuniones`.

Una buena minuta no transcribe todo. Captura:

- Contexto.
- Temas tratados.
- Acuerdos.
- Preguntas abiertas.
- Proximas acciones.

## Ritmo sugerido

- Revision semanal corta de propuestas y avances.
- Un espacio periodico para vision, portafolio y estrategia.
- Trabajo asincrono en issues, pull requests o documentos.

## Criterios de priorizacion

Al evaluar una iniciativa, miramos al menos:

- Potencial de generar valor economico.
- Velocidad para validar.
- Aprovechamiento de capacidades del equipo.
- Riesgo tecnico, legal, financiero y reputacional.
- Claridad del cliente, usuario o beneficiario.
- Posibilidad de aprender rapido aunque la idea cambie.

Para mayor detalle, ver [CRITERIOS-DE-OPORTUNIDAD.md](CRITERIOS-DE-OPORTUNIDAD.md).

## Principios no negociables

El equipo debe validar una lista corta de principios que funcionen como limites reales de decision. La version inicial esta en [PRINCIPIOS.md](PRINCIPIOS.md).

Una oportunidad que contradiga esos principios no deberia avanzar, incluso si parece rentable.
