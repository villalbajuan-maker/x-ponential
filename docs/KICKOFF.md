# Kickoff del repositorio

## Intencion

El kickoff busca convertir este repositorio en un espacio de participacion real. No se trata de presentar una version final al equipo, sino de abrir una primera base comun para discutir, ajustar y empezar a mover trabajo.

## Fecha de preparacion

2026-06-21

## Proxima reunion

2026-06-23

## Objetivos de la reunion

- Validar si el por que propuesto representa al equipo.
- Revisar los principios no negociables.
- Ajustar la lectura inicial de integrantes y roles.
- Acordar como usaremos GitHub para conversar, decidir y ejecutar.
- Elegir las primeras tareas antes de buscar oportunidades mas grandes.

## Antes de la reunion

Cada integrante deberia revisar:

- [PORQUE.md](PORQUE.md)
- [PRINCIPIOS.md](PRINCIPIOS.md)
- [EQUIPO.md](EQUIPO.md)
- [CRITERIOS-DE-OPORTUNIDAD.md](CRITERIOS-DE-OPORTUNIDAD.md)
- [TABLERO.md](TABLERO.md)

Y deberia abrir al menos una interaccion:

- Un issue de aporte personal.
- Un comentario sobre un principio.
- Una pregunta abierta.
- Una oportunidad inicial.
- Una correccion sobre su perfil.

## Agenda sugerida

| Bloque | Tiempo | Resultado esperado |
| --- | --- | --- |
| Apertura | 10 min | Recordar que el repo es punto de partida, no documento final |
| Por que | 20 min | Identificar acuerdos y tensiones |
| Principios | 25 min | Marcar principios aceptados, cuestionados o faltantes |
| Equipo | 20 min | Ajustar perfiles y posibles responsabilidades |
| GitHub | 15 min | Acordar uso minimo de issues, PRs y tablero |
| Proximos pasos | 20 min | Asignar primeras tareas con responsables |

## Regla de conversacion

Nada se corrige para ganar una discusion. Se corrige para que el equipo pueda ver mejor, decidir mejor y avanzar mejor.
