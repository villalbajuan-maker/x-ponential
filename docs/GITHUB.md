# Uso de GitHub

GitHub sera el espacio de memoria, discusion y seguimiento del equipo.

## Instrumentos

## Issues

Usaremos issues para abrir conversaciones concretas:

- Propuestas.
- Tareas.
- Preguntas abiertas.
- Discusion de principios.
- Ajustes al perfil del equipo.

Un buen issue debe tener contexto, pregunta o resultado esperado y un siguiente paso posible.

## Pull requests

Usaremos pull requests cuando alguien quiera modificar documentos, plantillas o codigo.

Un pull request debe explicar:

- Que cambia.
- Por que cambia.
- Que necesita revisar el equipo.
- Si contiene decisiones pendientes.

## Discussions

Si el repositorio se publica en GitHub y el equipo habilita Discussions, pueden usarse para conversaciones menos operativas:

- Vision.
- Lecturas de mercado.
- Preguntas filosoficas o estrategicas.
- Exploraciones que aun no son propuesta.

## GitHub Projects

Cuando el repo este conectado a GitHub, conviene crear un project board con estas columnas:

- Inbox
- Por revisar
- En discusion
- Decidido
- En ejecucion
- Bloqueado
- Cerrado

Mientras no exista el project board, [TABLERO.md](TABLERO.md) funcionara como tablero inicial.

## Reglas minimas

- Si una conversacion afecta al equipo, debe quedar en un issue, PR o documento.
- Si una decision cambia el rumbo, debe registrarse en `docs/decisiones`.
- Si una tarea tiene responsable, debe tener tambien siguiente paso.
- Si algo no esta claro, se abre una pregunta antes de inventar certeza.
